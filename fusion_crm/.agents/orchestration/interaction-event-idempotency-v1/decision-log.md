# Decision Log — interaction-event-idempotency-v1

- 2026-05-30 | Handoff: orchestrator/claude-code -> worker/claude-code for ENG-269.
- 2026-05-30 | Root cause: ingest services call create_event on every pull with no dedup → ×N duplication (measured ×5). raw_event is idempotent; the event layer is not.
- 2026-05-30 | Decision: dedup at the DB via partial UNIQUE INDEX on (tenant_id, source_provider, source_kind, source_external_id, kind) WHERE source_external_id IS NOT NULL; emission uses ON CONFLICT DO NOTHING. kind in the key so proposed→completed transitions are preserved.
- 2026-05-30 | Decision (append-only exception): the migration DELETEs duplicate interaction.event rows (keep earliest per key). This is a one-time migration-level data fix; the runtime service stays append-only (no update/delete service methods). Prod cleanup runs via the alembic Cloud Run Job on the next deploy.
- 2026-05-30 | Decision (savepoint, not session rollback): `InteractionService.create_event_idempotent` wraps the INSERT in `session.begin_nested()` (a SAVEPOINT) rather than calling `session.rollback()` on IntegrityError. CareStack ingest captures `ingest.raw_event` and emits `interaction.event` in the SAME outer transaction; a full session rollback would also undo the raw_event forensic capture, which we must preserve. The savepoint only undoes the failed INSERT — raw_event stays.
- 2026-05-30 | Decision (parallel idempotent API instead of return-type break): added `InteractionService.create_event_idempotent(...) -> EventEmissionResult(event, was_created)` while keeping `create_event(...) -> Event` as a thin wrapper. CareStack invoice/treatment/accounting callers switched to the new method so cross-pull conflicts count as "skipped" rather than "imported". Other callers (SF lead/event/task/opportunity/case, consultation_timeline) stay on `create_event` — they're either was_changed-gated or use `_find_existing_task_event` pre-checks, so conflicts are not their primary signal.
- 2026-05-30 | Verified on local DB: migration deleted 5026 duplicate rows (17158 → 12132). Per-kind counts now equal `count(distinct source_external_id)` across all 15 emitted kinds. `Collected` (sum of `payment_recorded.payload.amount`) is now $38,178.20 — exactly the deduped figure the mission goal called out. Migration round-trips clean (upgrade → downgrade -1 → upgrade head). Live `create_event_idempotent` re-emit returned `was_created=False`, the timeline count stayed at 1, and the savepoint preserved the in-flight `ingest.raw_event` row.
