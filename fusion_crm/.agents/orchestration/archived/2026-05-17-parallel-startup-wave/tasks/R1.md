## Implementation Requirements

- Add `NormalizedPersonHint` ORM model in the `ingest` schema.
- Add Pydantic DTOs needed for service/repository tests.
- Add repository methods:
  - `add_normalized_person_hint(...)`
  - `find_hint_by_raw_event(...)`
  - `list_unresolved_hints(...)`
- Add service method:
  - `capture_normalized_person_hint(...)`
- Keep the table tenant-scoped and additive.
- Keep PHI/raw payload out of normalized hint metadata. `meta` and
  `quality_flags` must be parser/matching metadata only; no clinical text and
  no verbatim provider payload.
- Do not import identity models or repositories. `person_uid` and
  `source_link_id` are plain UUID pointers.
- Use the P2 table shape unless an existing project pattern requires a small
  adjustment; document any adjustment in the report.

Expected table shape:

- `id`, `tenant_id`, `created_at`, `updated_at`
- `raw_event_id` with FK to `ingest.raw_event.id`
- `source_system` String(32)
- `source_kind` String(32)
- `source_id` String(240), nullable
- `observed_at` timezone datetime
- `given_name` String(120), nullable
- `family_name` String(120), nullable
- `display_name` String(240), nullable
- `email_normalized` String(320), nullable
- `phone_normalized` String(32), nullable
- `person_uid` UUID, nullable, plain pointer
- `source_link_id` UUID, nullable, plain pointer
- `payload_sha256` String(64), nullable
- `hint_hash` String(64), non-null
- `quality_flags` JSONB, non-null default `{}`
- `meta` JSONB, non-null default `{}`

Indexes/constraints:

- Prefer `UNIQUE (tenant_id, raw_event_id)` for this slice.
- `ix_normalized_person_hint_source` on
  `(tenant_id, source_system, source_kind, source_id)`.
- `ix_normalized_person_hint_email` on `(tenant_id, email_normalized)`.
- `ix_normalized_person_hint_phone` on `(tenant_id, phone_normalized)`.
- `ix_normalized_person_hint_person_uid` on `(tenant_id, person_uid)`.
- If source enum validation is added, keep it local to ingest and document how
  it will stay aligned with identity source lists.
