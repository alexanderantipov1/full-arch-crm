## Implementation Requirements

- Add `MatchCandidate` ORM model in `identity` schema.
- Add Pydantic DTOs only as needed for service/repository tests.
- Add repository methods for creating and reading match candidates by tenant.
- Add service-level validation for status, confidence, and accepted-person
  invariants. Keep PHI and raw provider payloads out of `evidence` and
  `conflicts`.
- Add one additive Alembic revision creating `identity.match_candidate`.
- Use tenant-scoped uniqueness/indexes per the P2 report:
  - status check: `open`, `auto_accepted`, `accepted`, `rejected`, `superseded`
  - confidence check in range 0..1
  - candidate/source self-match check
  - accepted-person check only for accepted statuses
  - indexes on candidate person, source person, hint, and status
  - partial unique open pair guard on `(tenant_id, person_pair_key)`
  - partial unique hint/candidate active guard on `(tenant_id, hint_id, candidate_person_uid)`
