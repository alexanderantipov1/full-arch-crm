## Implementation Requirements

- Bind the return value from `IngestService.capture(...)` and use the returned
  `RawEvent.id` when capturing the normalized person hint.
- Call `IngestService.capture_normalized_person_hint(...)` once per Salesforce
  record after raw capture and before identity resolution.
- Build `packages.identity.schemas.MatchHintIn` from the returned
  `NormalizedPersonHint` row; do not pass the raw SOQL record into identity.
- Call `IdentityService.resolve_or_create_from_hint(...)`.
- Fetch the resolved person with `IdentityService.get_person(...)`.
- Map `ResolveFromHintResult.was_existing_person_match` to
  `provider_metadata["is_reactivation"]`.
- Remove the hidden matching dependencies from the SF service:
  - no `self._identity._repo.find_source_link(...)`;
  - no `resolve_by_email(...)`;
  - no `resolve_by_phone(...)`;
  - no `add_source_link(...)`;
  - no `resolve_or_create_person(...)`.
- Keep the manual pull API and DTO shape unchanged.
- Preserve `pull_recent` limit validation, SOQL shape, raw capture before
  normalization, and `list_recent` behavior.
- Add a concise `packages/ingest/CLAUDE.md` note documenting the cutover
  pattern for future source handlers.

- hint capture uses the raw event id returned by `capture`;
- SOQL limit validation and list_recent mapping remain covered.
