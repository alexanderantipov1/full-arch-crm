## Implementation Requirements

- Add DTOs in `packages/identity/schemas.py`:
  - `MatchHintIn`: minimal identity-owned input DTO. It must not import from
    `packages.ingest`. Include at least `hint_id`, `source_system`,
    `source_kind`, `source_id`, `given_name`, `family_name`, `display_name`,
    `email_normalized`, `phone_normalized`, `quality_flags`, and `meta`.
  - `ResolveFromHintResult`: include `person_uid`, `was_new_person`,
    `was_existing_person_match`, `was_source_link_recapture`,
    `match_candidate_id`, and `open_candidate`.
- Add a match rule taxonomy in `packages/identity/models.py`, for example:
  `source_link`, `email_phone_name`, `phone_name`, `email_name`,
  `email_only_ambiguous`, `phone_only_ambiguous`.
- Add service-level validation so `MatchCandidateIn.match_rule` is one of the
  approved rules.
- Add repository helper(s) needed to list candidate persons by normalized
  email/phone identifiers. All reads must be tenant-scoped through
  `for_tenant(...)`; no cross-tenant fallback.
- Add `IdentityService.resolve_or_create_from_hint(...)` with this tier ladder:
  - Tier 0: exact `(source_system, source_kind, source_id)` source link
    recapture. Touch the source link, return the existing person, and do not
    write a match candidate.
  - Tier 1: high-confidence existing-person match. Add the new source link to
    that person, write `MatchCandidate(status="auto_accepted")`, and return
    `was_existing_person_match=True`.
  - Tier 2: ambiguous match. Create a new source-linked person, write
    `MatchCandidate(status="open")` with conflict evidence, and return
    `open_candidate=True`.
  - Fallback: create a new source-linked person and return
    `was_new_person=True`.
- Use the initial thresholds from R2/P2:
  - email + phone + compatible name: `0.99`
  - phone + compatible name: `0.95`
  - email + compatible name with no phone conflict: `0.92`
  - email-only or phone-only ambiguity: below auto-accept, open candidate
- Keep evidence/conflict payloads normalized and PHI-free. Reuse the recursive
  PHI/raw-payload guard already used by `add_match_candidate(...)`.
- Preserve idempotency:
  - source-link recapture does not write a match candidate;
  - re-calling with the same `hint_id` and candidate person should reuse or
    avoid duplicating the active candidate row.
- Do not import `packages.ingest` from identity.
