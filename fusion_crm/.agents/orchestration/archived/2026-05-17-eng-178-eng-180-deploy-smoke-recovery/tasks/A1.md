Known defect:
- In `.github/workflows/deploy-prod.yml`, `BODY=$(check "/healthz")` captures stdout from `check()`.
- On failure, diagnostic `echo` lines and `fail()` output are swallowed by the command substitution, so GitHub Actions shows only a generic exit.
- The intended fix is to send diagnostics to stderr while preserving response body on stdout.

Shared contract:
- Successful `check()` calls must still print only the response body on stdout.
- Failure diagnostics must be visible in GitHub Actions logs.
- Do not change endpoint coverage, expected status codes, rollback behavior, IAP audience, or deployment semantics.
