# Invoice Sync

**Fusion domain:** `billing`
**PHI:** mixed — invoice rows are financial, but `patientId` ties them to a specific patient, so we treat the whole feed as PHI-adjacent.
**Spec section:** Sync Resource 5 (CareStack Developer API v1.0.45)

## Endpoint

### `GET /v1.0/sync/invoices` — modified-after invoice feed

Real path: `GET /api/{version}/sync/invoices`.

- **Query params:**
  - `modifiedSince` (datetime, ISO 8601 UTC) — required on the initial call.
  - `continueToken` (string, optional).
  - `pageSize` (number, optional) — default `100`, max `500`.
- **Response:** list of Invoice objects plus a `continueToken`. Token is `null` when the scan is complete.
- **Scope:** covers all invoices generated by the practice, including unapplied ones and advance payment invoices. For advance-payment invoices the response includes the corresponding provider; otherwise that field is `null`. Unapplied credits are included in the corresponding response field.

## Fields returned (summary)

| field | type | notes |
|---|---|---|
| invoiceId | integer | natural key |
| amount | decimal | |
| unappliedAmount | decimal | remaining credit |
| providerId | integer, optional | populated for advance-payment invoices |
| locationId | integer | FK |
| isDeleted | bool | tombstone flag |
| patientId | integer, optional | PHI linkage when present |
| lastUpdatedOn | datetime | watermark source |
| paymentCategory | string | see enum below |
| invoiceNumber | integer | practice-facing number |
| paymentDate | datetime | |
| invoiceType | integer | 1 Regular, 2 Advance, 3 Capitation, 4 PaymentPlan |
| invoiceSource | integer | 1 PMS, 2 TextToPay, 3 PatientPortal, 4 OnlineAppointment, 5 Subscription, 6 PaymentLink |
| paymentTypeId | integer | FK |
| isNsf | bool | "not sufficient funds" flag |

### `paymentCategory` enum

`TRANSFER`, `CASH`, `CHECK`, `CREDIT/DEBIT CARD`, `THIRD PARTY FINANCING`,
`DIRECT TRANSFER`, `SPECIAL CREDIT`, `INCOME REDUCTION`, `MEMBERSHIP CREDIT`.

## Fusion mapping

- Target table(s): `billing.invoice` (new).
- `ingest.raw_event.event_type`: `carestack.invoice.upsert`.
- Cadence: every 15 minutes.
- Idempotency key: `(invoiceId, lastUpdatedOn)`.
