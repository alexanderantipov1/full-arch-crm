API Reference​
     ●​ CareStack exposes REST API over HTTPS to integrate with other systems.
     ●​ In general, the CareStack API uses HTTP requests with JSON arguments, returns JSON
        responses, and uses standard HTTP response codes and verbs.


Version​
     ●​ The version to be used in the API endpoint is a combination of the “MAJOR” and
        “MINOR” version numbers.
     ●​ In this instance, the “{version}” in the endpoints should be replaced with “v1.0”.


Authentication
The CareStack API uses token based authentication for accessing API’s endpoints. The token is
issued by CareStack’s IdentityProvider(Idp) service, the details given below.

Prerequisites
A third party client will be provided the below details from CareStack support after successful
registration process.

           Base url               Base url of identity provider (eg: https://id.carestack.com)
           ClientID               A client id unique to the third-party vendor for token request.
           ClientSecret           A secret key for the client for the token request.
           Username               A key used as the user name for the token request, also known as vendor key.
                                  This will be the same for a vendor.
           Password               A key used as password for obtaining access token for a specific CareStack
                                  account, also known as account key. This key will be different for each account.


Token Request endpoint
Make a POST request to <base url>/connect/token passing the below parameters as
application/x-www-form-urlencoded content type. A response status of 200 will provide a
successful access token, which requires to be passed as Authorisation Bearer token in requests
to API endpoints.
CareStack Developer API v1.0.40
Aug 04, 2025


           grant_type             “password”
           client_id              <client id obtained after registration>
           client_secret          <client secret obtained after registration>
           username string        <username obtained after registration>
           password string        <password specific to the CareStack Account>


Note: When the access token expires, it fails with a 401 error status .A new access token has to
be obtained for continued access. The access token has a duration of I hour and can be obtained
from the “iat” and “exp” attributes in the JWT.


Errors​
     ●​ HTTP response status codes are used to indicate the success or failure of the API
        requests.
     ●​ In general, status codes in 2xx are used for success, 4xx for client errors and 5xx for
        server errors.
     ●​ Refer to section “Error Response Signatures” for more information on handled error
        codes.

Usage of communicationStatus
    ●​ The communicationStatus is set using a bitwise OR logic.
    ●​ For instance if you need a patient who accepts Promotion(8) and Text(2).
    ●​ Value to be passed for communicationStatus is, bitwise OR operation of 8 and 2, which gives you 10.

Sample code for AES256 in C#

private const short IV_SIZE = 16;

   public byte[] Encrypt(byte[] data, byte[] encryptionKey)
   {
     using AesCryptoServiceProvider aesCryptoServiceProvider = new
AesCryptoServiceProvider();
     aesCryptoServiceProvider.Key = encryptionKey;
     byte[] array =
aesCryptoServiceProvider.CreateEncryptor(aesCryptoServiceProvider.Key,
aesCryptoServiceProvider.IV).TransformFinalBlock(data, 0, data.Length);
     byte[] array2 = new byte[16 + array.Length];
     Buffer.BlockCopy(aesCryptoServiceProvider.IV, 0, array2, 0, 16);
     Buffer.BlockCopy(array, 0, array2, 16, array.Length);
     return array2;
   }

   public byte[] Decrypt(byte[] data, byte[] key)
   {
     using AesCryptoServiceProvider aesCryptoServiceProvider = new
AesCryptoServiceProvider();
     aesCryptoServiceProvider.Key = key;
     byte[] array = new byte[16];
     Buffer.BlockCopy(data, 0, array, 0, 16);
     aesCryptoServiceProvider.IV = array;
     return aesCryptoServiceProvider.CreateDecryptor(aesCryptoServiceProvider.Key,
aesCryptoServiceProvider.IV).TransformFinalBlock(data, 16, data.Length - 16);
   }
