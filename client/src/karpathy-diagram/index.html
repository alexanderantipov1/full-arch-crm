♻️ THE FLYWHEEL — Why the system gets smarter over time
More clinics connect → more anonymized patterns flow into wiki → agents make better decisions → higher claim acceptance rates → better treatment outcomes → more clinics join. Each new DSO makes the system smarter for all existing DSOs. No PHI ever enters the wiki.
🔒 HIPAA COMPLIANCE LAYER
DatabaseAdapter enforces PHI isolation at the boundary. All wiki ingests use dataClass: 'ops_safe' — never 'phi'. Raw patient records (name, DOB, SSN, chart #) never leave the tenant's adapter scope. Wiki pages contain only aggregated, anonymized statistics and pattern language.
📥
INGEST
Called after every clinical event. The LLM reads the existing wiki page for that topic, plus the new event data, and synthesizes an updated summary — appending new patterns, updating statistics, noting the timestamp.
TRIGGER POINTS
📄 eob-service.ts → after every EOB line
🔬 engine.ts → after every runBatch()
🔌 Any adapter event (appointment, treatment plan)
await wikiService.ingest(
  { type: 'EOB_DENIED',
    cdtCode: 'D6010',
    denialReason: 'missing_PA',
    payer: 'delta_dental' },
  'ops_safe'
);
🔍
QUERY
Agents call query() before every important decision. WikiService does a semantic search across all relevant pages, then synthesizeAnswerAsync() uses Claude to produce a grounded, context-aware answer from the wiki content.
CALLER POINTS
🎯 orchestrator.ts → before runCycle()
💼 InsuranceAgent → before filing claim
🦷 TreatmentPlanAgent → before case presentation
const answer = await
  wikiService.query(
    "What causes D6010 denials
     at Delta Dental?",
    'ops_safe'
  );
// → "Missing prior auth is
//    the #1 cause (67% of denials)..."
🧹
LINT
Weekly automated pass over all wiki pages. Removes entries older than 90 days that haven't been re-confirmed, resolves contradictions (newer data wins), merges duplicates, and refreshes KPIs from live database statistics.
LINT OPERATIONS
🗑️ Remove stale entries (>90 days, unconfirmed)
⚖️ Resolve contradictions (newer data wins)
🔀 Merge duplicate pattern entries
📊 Refresh KPIs from live DB stats
// Scheduled weekly (cron)
await wikiService.lint();
// → Removed: 3 stale entries
// → Merged: 2 duplicates
// → Refreshed: 18 KPI values
// → Wiki health: 97%
